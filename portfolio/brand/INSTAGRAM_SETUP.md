## 4. Content pillars

Realistic plan for a CS student targeting internships and co-ops. Post **1-2 times per week**; stories can be more casual (2-4x/week when busy is fine).

### Pillar 1 — Project proof (40%)

- Portfolio screenshots and case study carousels
- Short reels: scroll through a deployed site or API docs
- Before/after or "what I built" slides (weROI, PNTCOG, StudySync)

### Pillar 2 — Build logs (25%)

- "Week in code" stories: commits, PRs, bug fixes
- Terminal or IDE clips with voiceover or text overlay
- Deployment wins (Vercel, Railway)

### Pillar 3 — Student + fundamentals (20%)

- Coursework connections (data structures, databases, networking) without sharing graded work
- OWASP / PortSwigger lab milestones (concept, not screenshots of answers)
- Dean's List and CSEC wins **occasionally**, not every post (supporting proof, not the headline)

### Pillar 4 — Tech tips (10%)

- One tip per carousel: Git, REST design, Tailwind, FastAPI patterns
- Repurpose portfolio case study "what I learned" bullets

### Pillar 5 — Open to opportunities (5%)

- Quarterly "open to internships" post or story
- Pin or highlight when actively applying

### Story ideas (low effort)

- Poll: "React or vanilla JS for this project?"
- Quiz sticker: "What does JWT stand for?"
- Link sticker to portfolio project pages

## 6. Aesthetic

Match the portfolio obsidian / electric indigo system from `app/globals.css`.

### Color palette

| Token | Hex | Use |
|-------|-----|-----|
| Obsidian | `#050505` | Backgrounds, story covers |
| Charcoal | `#151515` | Card slides |
| Electric indigo | `#1500ff` | Accents, headings, links |
| Paper | `#fdfdfd` | Primary text on dark |
| Pearl | `#d4d4d4` | Secondary text |
| Ash | `#a3a3a3` | Captions, meta |
| StudySync accent | `#6366f1` | Optional per-project slides |

### Visual rules

- **Dark edits default:** screenshots on obsidian frames, slight border `#2f2f2f`
- **Typography:** DM Sans (Google Fonts in Canva) or Inter; one font family across grid
- **Consistent templates:** 1080x1080 feed, 1080x1920 stories/reels
- **Project accents:** use per-project colors from portfolio (weROI indigo, PNTCOG gold) sparingly

### Grid preview strategy

1. **Row 1 (top):** strongest visuals: portfolio hero, StudySync, GitHub ( varied but all dark-framed )
2. **Row 2:** text-forward internship post centered among two project shots
3. **Row 3+:** alternate **photo/screenshot** and **typography** posts to avoid a muddy all-screenshot grid
4. Preview in Canva **Instagram planner** or Preview app before publishing
5. Avoid bright white screenshots without a dark border; they break the grid flow

### Reels style

- 9:16 crop of screen recordings
- Subtitles on (muted scrollers)
- 15-30 seconds: hook in first 2 seconds ("Built this API in FastAPI")

## 7. What NOT to post

| Avoid | Why |
|-------|-----|
| weROI source code, env files, admin credentials, internal dashboards | Security and client trust |
| Private client details, contract terms, payment drama, unreleased features | Unprofessional; legal risk |
| Framing weROI as an **active contract** you are still delivering under | Truthful framing: platform build experience; say "built" not "currently contracting" unless true |
| Fake follower counts, inflated metrics, or "500% growth" claims | Hurts credibility with recruiters |
| Graded exam content, plagiarism-adjacent coursework uploads | Academic integrity |
| Hot takes, politics, or unrelated personal drama | Keep feed hireable |
| CSEC / Dean's List on **every** post | Use as proof occasionally, not the whole brand |
| Begging for internships in comments on unrelated viral posts | Use dedicated posts and LinkedIn instead |

**Safe weROI language:** "Built a full-stack agency platform" / "Production web app experience" / "Case study on my portfolio"

**Avoid:** "My client at weROI" / "Active agency contract" (unless factually current and approved)
